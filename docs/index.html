Introduction

What Gungnr is

Gungnr is a dockerized control panel for deploying template-based projects and exposing services through a Cloudflare Tunnel. It runs on the host and uses Docker, local templates, and a user-managed cloudflared tunnel.

Scope and guarantees

  • Install once. Bootstrap wires OAuth, Cloudflare, and Docker before the panel launches.
  • The panel starts only when prerequisites are verified.
  • deploy.sh is reference-only; the UI mirrors its behavior.
  • Cloudflared runs on the host as a user-managed process.

Architecture overview

Host

Runs Docker, stores templates, and executes cloudflared locally.

Docker Compose

Boots API, UI, and Postgres using the generated .env.

Cloudflare Tunnel

Routes https://panel.domain to the local Nginx proxy on port 80.

GitHub OAuth

Controls login and allowlisted access to the panel.

Installation

Bootstrap the host

The installer verifies dependencies. The bootstrap CLI creates a tunnel, configures DNS, writes environment variables, and starts the stack.

Prerequisites

  • Sudo access on the host machine.
  • Cloudflare account with a domain already added.
  • Cloudflare API token with Tunnel edit and DNS edit permissions.
  • GitHub OAuth app credentials (Client ID and Client Secret).
  • GitHub callback URL set to https://panel.your-domain/auth/callback.
  • GitHub OAuth app has Enable Device Flow toggled on for CLI bootstrap.
  • Access to the host network for outbound Cloudflare requests.

Official install docs

  • Cloudflared downloads
  • GitHub CLI installation
  • Docker Engine install (Ubuntu)
  • Docker Compose plugin install

Step 01

Run the installer

Downloads and runs install.sh to verify dependencies. 

wget -qO /tmp/gungnr-install.sh https://raw.githubusercontent.com/Hrafngud/gungnr/main/install.sh && chmod +x /tmp/gungnr-install.sh && /tmp/gungnr-install.sh

Step 02

Run bootstrap

The CLI prompts for GitHub, Cloudflare, and DNS inputs.

Ensure your GitHub OAuth app has Enable Device Flow toggled on before continuing. 

gungnr bootstrap

Step 03

Open the panel

The CLI prints the panel URL after the health check succeeds. 

https://panel.your-domain

Usage

Operate the panel confidently

Once bootstrap completes, the panel is ready. The key flows are listed here so you can run daily operations without digging through UI menus.

First login

  • Open the panel URL printed by bootstrap.
  • Log in with GitHub. The bootstrap user is seeded as SuperUser.
  • Add additional users from the allowlist in the panel if needed.

Deploy flows

  • Create from template: uses the GitHub App configuration.
  • Deploy existing: forwards a local project to a subdomain.
  • Quick local service: expose any host port with a subdomain.

Logs + status

  • Use the Jobs page to monitor deployments and tunnel changes.
  • Container logs stream under Host Settings.
  • Audit log records each system action.

Operations commands

Run these from the repo root to manage the stack directly. 

make up
make up-d
make logs
make down
make down-v

Errors

Troubleshooting

These fixes map to real installer and bootstrap error messages. Start with triage commands, then drill into the matching error below.

Quick triage

docker compose ps
make logs
cloudflared tunnel list
cloudflared tunnel info <tunnel-uuid>
cloudflared tunnel ingress validate
cloudflared tunnel ingress rule https://panel.your-domain
docker not found in PATH. Install Docker and retry

Docker is missing or not running. Install Docker Engine and ensure docker info works. Restart the Docker service and re-run the installer.

docker compose not available. Install Docker Compose v2 (docker compose) or docker-compose

Upgrade Docker or install the Compose plugin. Run docker compose version to verify.

docker-compose.yml not found from ... upward; run bootstrap from the repo root

Run gungnr bootstrap from the repo root so the CLI can find docker-compose.yml.

cloudflared not found in PATH. Install cloudflared and retry

Install cloudflared and confirm cloudflared --version succeeds.

cloudflared tunnel login failed

Re-run cloudflared tunnel login manually and complete the browser auth. Confirm the correct Cloudflare account is selected.

cloudflared credentials not found after login

The CLI expects ~/.cloudflared/cert.pem. Confirm the file exists and is readable. Use the same directory during login and bootstrap.

cloudflared tunnel credentials not found after create

The tunnel JSON should appear under ~/.cloudflared. Check permissions and retry tunnel creation.

cloudflared tunnel run failed

Check ~/.cloudflared/cloudflared.log for errors. Verify the tunnel ID exists and the config points to the correct credentials file.

Cloudflare zone lookup failed / no active zone found

Ensure the domain is added to Cloudflare and the token has Zone read access. Use the root domain (example.com), not a subdomain.

Cloudflare DNS lookup failed / DNS record not found after routing

Confirm the token includes DNS edit permissions. Re-run cloudflared tunnel route dns <tunnel-uuid> panel.example.com and check the DNS record in Cloudflare.

tunnel did not report active connections before timeout

The tunnel process did not report connections. Run cloudflared tunnel info, verify outbound network access, and confirm the process is running.

timed out waiting for API health / health check returned 500

The stack started but the API did not become healthy. Check make logs, confirm Postgres is up, and ensure port 80 is free on the host.

template catalog not configured / template creation disabled

Configure the GitHub App in Host Settings. Template creation is disabled until App ID, client credentials, installation ID, and private key are set.

login loop or access denied after GitHub login

Confirm the user is on the allowlist. The bootstrap user is seeded as SuperUser, but additional users must be added in the panel.
