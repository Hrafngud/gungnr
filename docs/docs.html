Introduction

What Gungnr is

Gungnr is a dockerized control panel for deploying template-based projects and exposing services through a Cloudflare Tunnel. It runs on the host and uses Docker, local templates, and a user-managed cloudflared tunnel.

Architecture overview

Host

Runs Docker, stores templates, and executes cloudflared locally.

Docker Compose

Boots API, UI, and Postgres using the generated .env.

Cloudflare Tunnel

Routes https://panel.domain to the local Nginx proxy on port 80.

GitHub OAuth

Controls login and allowlisted access to the panel.

Installation

Bootstrap the host

The installer verifies dependencies. The bootstrap CLI creates a tunnel, configures DNS, writes environ ment variables, and starts the stack.

Prerequisites

  • Sudo access on the host machine.
  • Cloudflare account with a domain already added.
  • Cloudflare API token with Tunnel edit and DNS edit permissions.
  • GitHub OAuth app credentials (Client ID and Client Secret).
  • GitHub callback URL set to https://panel.your-domain/auth/callback.
  • GitHub OAuth app has Enable Device Flow toggled on for CLI bootstrap.
  • Access to the host network for outbound Cloudflare requests.

Official install docs

  • Cloudflared downloads
  • GitHub CLI installation
  • Docker Engine install (Ubuntu)
  • Docker Compose plugin install

Compatibility

Currently, Gungnr is only supported on Linux, we are looking foward to introduce a compatibility layer for other OS soon:

  • Linux (amd64/arm64) with apt, dnf, yum, pacman, apk, or zypper.

Step 01

Set up external services

Gungnr relies on external services for authentication, tunneling and DNS management. We strongly recommend configuring GitHub OAuth app and Cloudflare credentials before running the installer. This will ensure a blazingly fast bootstrap experience.

GitHub OAuth Setup

Create a GitHub OAuth app for panel authentication.

1. Create OAuth App

Open GitHub Developer Settings and click New OAuth App to register the panel. Those will be used as identity provider and RBAC for the panel, through allowlists.

2. Configure URLs

On the OAuth App form in GitHub Developer Settings , set the Homepage URL to your panel base URL and set the Authorization callback URL to https://panel.your-domain/auth/callback. These must match the domain that will front the panel.

3. Enable Device Flow

After the app is created, open it from the OAuth Apps list in GitHub Developer Settings and enable Device Flow so the CLI bootstrap can authenticate without a browser session.

4. Save Credentials

On the same OAuth App settings page in GitHub Developer Settings , copy the Client ID, then generate a Client Secret and store it safely for the bootstrap prompts.

Cloudflare Setup

Configure Cloudflare for tunnel routing and DNS management.

1. Add Domain

Log in to the Cloudflare dashboard and use Onboard a domain to add your apex domain. Update your registrar nameservers so the zone becomes Active.

2. Create API Token

Open API Tokens in your Cloudflare profile and click Create Token.

3. Grant Permissions

During token setup in API Tokens , scope the token to the account that will host the panel and scope to all zones. Aditionally, scope to all accounts to fetch authorized zones from other Cloudflare accounts you're admin if needed. Confirm the token can correct Scopes listed here:

  • All accounts
    • Registrar: Domains — Read
    • DNS Views — Edit
    • DNS Settings — Edit
    • Cloudflare Tunnel — Edit
  • All zones
    • DNS Settings — Edit
    • Zone — Read
  • All users
    • API Tokens — Read

4. Save Token

Copy the token value and store it safely, before leaving the API Tokens page. You will paste it during bootstrap, and it is only shown once.

Step 02

Run the installer

Downloads and runs install.sh to verify dependencies. 

wget -qO /tmp/gungnr-install.sh https://raw.githubusercontent.com/Hrafngud/gungnr/main/install.sh && chmod +x /tmp/gungnr-install.sh && /tmp/gungnr-install.sh

Step 03

Run bootstrap

The CLI prompts for GitHub, Cloudflare, and DNS inputs.

Ensure your GitHub OAuth app has Enable Device Flow toggled on before continuing. 

gungnr bootstrap

Step 03

Open the panel

The CLI prints the panel URL after the health check succeeds. 

https://panel.your-domain

Tunnel auto-start watchdog

Bootstrap installs a lightweight cron watchdog so the tunnel restarts on reboot and is checked every 5 minutes. The scripts live under ~/gungnr/state.

  • ~/gungnr/state/cloudflared-run.sh (starts the tunnel with the generated config)
  • ~/gungnr/state/cloudflared-ensure.sh (verifies the process and relaunches if needed)
~/gungnr/state/cloudflared-ensure.sh
crontab -l | rg 'gungnr-cloudflared'
crontab -l | rg -v 'gungnr-cloudflared' | crontab -

Usage

Operate the panel confidently

Once bootstrap completes, the panel is ready. The key flows are listed here so you can run daily operations without digging through UI menus.

First login

  • Open the panel URL printed by bootstrap.
  • Log in with GitHub. The bootstrap user is seeded as SuperUser.
  • Add additional users from the allowlist in the panel if needed.

    • OBS: Gungnr currently have a simple RBAC implementation where SuperUsers can manage everything, Admins have quite same privileges, but can't assign roles, and normal users can perform deployments and jobs, but can't add people to the allowlist. This feature is planned to be expanded in future releases. But you can feel free to fork and customize the middleware permissions to your personal needs.

Deploy flows

  • Create from template: uses the GitHub App configuration to start a fresh project from a curated fullstack template. Currently we only feature a Vue + Go + PostgreSQL opinated template, but further, new templates will be added with different stacks.
  • Deploy existing: forwards a local application running on localhost:PORT to a subdomain. Useful to front-end quick view from multiple devices
  • Quick services: Pulls a docker registry official image/compose and fowards to a tunnel. You can do some quite intresting setups quickly, test services

Logs + status

  • Use the Jobs page to monitor deployments and tunnel changes.
  • Container logs stream under Host Settings.
  • Audit log records each system action.

CLI commands

Gungnr has a few cli commands available, those can be used to controll the compose, tunnel and overall states

Remove containers and restart compose: 

                  gungnr restart
Run the cloudflared tunnel: 

                  gungnr tunnel run
Uninstall 

                  gungnr uninstall

We are looking foward to introduce new CLI methods in the future
